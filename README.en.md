## üèÜ Principles I Follow: SOLID

As a developer, I follow the **SOLID** principles to write clean, maintainable, and scalable code:

- **S** - Single Responsibility Principle: Each class has a single responsibility
- **O** - Open/Closed Principle: Open for extension, closed for modification
- **L** - Liskov Substitution Principle: Object substitution without breaking the system
- **I** - Interface Segregation Principle: Specific and lean interfaces
- **D** - Dependency Inversion Principle: Depend on abstractions, not implementations
